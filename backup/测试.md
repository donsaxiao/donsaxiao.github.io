在 Windows 10 中放行端口（如8080端口），本质上是需要在“Windows Defender 防火墙”中创建一条**入站规则**。我为你整理了两种最常用的方法：一种是通过图形界面操作（适合大多数用户），另一种是通过命令行操作（适合高级用户或批量处理）。

## 🖱️ 方法一：图形界面操作（推荐）

这是最直观的方法，通过控制面板或直接运行命令打开防火墙设置：

1.  **打开高级安全设置**
    *   按下键盘上的 Win + R 键，输入 wf.msc 并回车。这将直接打开“高级安全 Windows Defender 防火墙”。
    *   *替代方法：* 你也可以通过“控制面板” -> “系统和安全” -> “Windows Defender 防火墙” -> 点击左侧的“高级设置”进入。

2.  **新建入站规则**
    *   在打开的窗口中，点击左侧的**“入站规则”**。
    *   在右侧的操作栏中，点击**“新建规则...”**。

3.  **配置规则参数**
    *   **规则类型：** 选择**“端口”**，点击“下一步”。
    *   **协议和端口：**
        *   选择 **TCP**。
        *   选择“特定本地端口”，输入 **8080**。
        *   *(如果你需要开放 UDP 端口，或者一个范围如 8000-9000，也可以在此处选择)*。
    *   **操作：** 选择**“允许连接”**。

4.  **选择应用范围与命名**
    *   **配置文件：** 建议勾选**“域”、“专用”、“公用”**（全选），以确保在各种网络环境下端口都开放。
    *   **名称：** 输入一个好记的名称，例如 Allow Port 8080，可以加个描述。
    *   点击“完成”。

## ⌨️ 方法二：命令行操作（快速）

如果你喜欢使用命令行，或者需要快速执行，可以使用以下两种方式：

### 方式 A：使用 CMD（管理员）
1.  按 Win + X，选择“Windows 终端(管理员)”或“命令提示符(管理员)”。
2.  输入以下命令并回车执行：

```cmd
 netsh advfirewall firewall add rule name="Open Port 8080" dir=in action=allow protocol=TCP localport=8080
```
### 方式 B：使用 PowerShell（管理员）
1.  同样以管理员身份运行 PowerShell。
2.  输入以下命令并回车执行：

```cmd
New-NetFirewallRule -DisplayName "Allow 8080" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 8080
```

## 🛡️ 进阶设置：仅允许特定 IP 访问（更安全）

如果你是在局域网内开发调试，希望只有特定的同事或设备能访问你的 8080 端口，可以增加 IP 限制：

1.  在“新建规则”向导中，**不要**选择“端口”，而是选择**“自定义”**。
2.  在“协议和端口”步骤，设置 TCP 和本地端口 8080。
3.  在**“作用域”**步骤：
    *   在“下列远程 IP 地址”中，点击“添加”。
    *   输入允许访问的设备 IP 地址（例如同事的电脑 IP）。
4.  继续完成后续的“允许连接”和命名步骤。

## ✅ 验证是否成功

设置完成后，你可以通过以下方式简单测试端口是否已开放：

1.  **本机测试：** 打开浏览器访问 http://127.0.0.1:8080，看你的服务是否能响应。
2.  **Telnet 测试：** 如果你在本机安装了 Telnet 客户端，可以在 CMD 中输入 telnet 127.0.0.1 8080，如果屏幕变黑或出现空白，说明端口是通的。
3.  **局域网测试：** 让局域网内的另一台电脑访问 http://你的IP地址:8080。

## 💡 特别提醒

*   **服务监听地址：** 有时候端口放行了但依然无法访问，可能是因为你的程序（如 Node.js、Tomcat）只监听了 127.0.0.1。请确保你的服务监听的是 0.0.0.0，这样才能接收外部请求。
*   **路由器/安全组：** 上述步骤仅解除了电脑本地的防火墙。如果你希望**公网**（外网）访问你的 8080 端口，还需要在路由器上做**端口映射**（Port Forwarding）；如果你是在云服务器（如阿里云、腾讯云）上操作，则需要在云平台的“安全组”中放行该端口。
